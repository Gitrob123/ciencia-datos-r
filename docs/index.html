1 🌲 Random Forest en R

1.1 1️⃣ ¿Qué es Random Forest?

Random Forest es un algoritmo de Machine Learning supervisado basado en:

  • 🌳 Muchos árboles de decisión
  • 🎲 Muestreo aleatorio (bootstrap)
  • 🎯 Votación mayoritaria (clasificación)
  • 📊 Promedio (regresión)

Pertenece a los métodos Ensemble, es decir, combina múltiples modelos débiles para crear uno fuerte.

1.1.1 ¿Por qué funciona tan bien?

Porque:

  • Reduce el overfitting de un árbol individual
  • Reduce la varianza
  • Es robusto al ruido
  • Maneja muchas variables
  • Calcula importancia de variables automáticamente

5 5️⃣ División entrenamiento y prueba

Dividimos en 70% entrenamiento y 30% prueba.

set.seed(123)

trainIndex <- createDataPartition(iris$Species, 
                                   p = 0.7, 
                                   list = FALSE)

train <- iris[trainIndex, ]
test  <- iris[-trainIndex, ]

6 6️⃣ Entrenamiento del modelo Random Forest

set.seed(123)

modelo_rf <- randomForest(Species ~ ., 
                          data = train,
                          ntree = 500,
                          mtry = 2,
                          importance = TRUE)

11 1️⃣1️⃣ ¿Cómo funciona internamente?

11.1 Paso 1: Bootstrap

Cada árbol usa una muestra aleatoria con reemplazo.

11.2 Paso 2: Selección aleatoria de variables

En cada nodo:

  • Se eligen aleatoriamente mtry variables
  • Se busca la mejor división entre esas

11.3 Paso 3: Votación

Clasificación: - Gana la clase con más votos

Regresión: - Se promedian los valores

12 1️⃣2️⃣ Ventajas y desventajas

12.1 Ventajas

✔ Alta precisión
✔ Maneja datos no lineales
✔ No requiere normalización
✔ Calcula importancia

12.2 Desventajas

✘ Puede ser más lento
✘ Menos interpretable que un árbol simple

13 1️⃣3️⃣ Comparación con Árbol de Decisión

Un árbol solo:

  • Alta varianza
  • Se sobreajusta

Random Forest:

  • Reduce varianza
  • Generaliza mejor
